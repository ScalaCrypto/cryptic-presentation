Keeping

Top-Down

Bottom-Up

  • There are different approaches you can use to keep secrets
  • You can either encrypt the whole database or filesystem
  • This is a → Top-Down approch
  • Or only encrypt the sensitive parts
  • This is a → Bottom-Up approach
  • With Top-Down - you have few keys, each used widely
  • With Bottom-Up - you have many keys, each used narrowly

Forgetting

GDPR

Backups

Throw away keys

  • So, what about forgetting?
  • Why do we need to forget secrets?
  • → GDPR can require us to forget
  • With a Top-Down approach you have the problem of how to handle → backups containing a wide set of data using only a few keys
  • With a Bottom-Up approach - you can instead → throw away the specific key used for the narrow set of data to be forgotten

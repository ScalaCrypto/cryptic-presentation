Encryption Approaches

 

Top-Down

Few keys, wide dataset

 

Bottom-Up

Many keys, narrow dataset

  • There are basic 2 different appr you can use when encrypting your data
  • You can either enc on the level of an entire db-table or even an entire fs
  • This can be called a → Top-Down appr: using this appr likely leads to few keys, each used on wide datasets →
  • Or u can enc only the sensi parts of u data
  • This can be called a → Bottom-Up appr
  • Here u can have many ks, e used on narrow dataset →

Forgetting

GDPR

Backups

Throw away keys

  • So, what about forgetting?
  • Why do we need to forget secrets?
  • → Well, GDPR can require us to forget
  • With a Top-Down approach you have the problem with backups → how do you forget a specific user?
  • With a Bottom-Up approach - you can instead → throw away that users key
